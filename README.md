## 🧠 Опис функціоналу

- 🛡️ **AI Gateway (Security & Telemetry)**: Проксі-сервер, що стоїть між клієнтом та LLM. Блокує атаки типу Prompt Injection (за 0-6 мс) за допомогою RegEx-правил, а також збирає точні метрики затримок (TTFT, TPOT).
- 🔗 **MCP Server (Twenty CRM)**: Сервер, який надає LLM доступ до корпоративної бази даних Twenty CRM через стандартизований Model Context Protocol. Дозволяє ШІ "читати" та "змінювати" дані користувачів за запитом.
- 📊 **Автоматизований Бенчмаркінг**: Набір скриптів (`run_benchmark.js`) для навантажувального тестування моделей за 5 категоріями (Short, Reasoning, MCP, Context-Heavy, Security).
- 📈 **Візуалізація Даних (Data Science)**: Python-скрипти для автоматичної побудови професійних академічних графіків (Boxplots, Scatterplots) на основі зібраних логів.
- 💬 **Клієнтський Інтерфейс**: Інтеграція з Open WebUI для зручного доступу користувачів до ШІ-асистента.

## 🛠️ Ендпоінти MCP Сервера (Інструменти)

MCP-сервер надає великим мовним моделям наступні інструменти (Tools) для взаємодії з Twenty CRM:

- **`get_my_clients`** — Отримання списку всіх клієнтів, до яких у поточного користувача є доступ (власні клієнти + клієнти підлеглих) згідно з Role-Level Security (RLS).
- **`get_client_details`** — Отримання детальної інформації про клієнта та історії зустрічей (нотаток) за його ім'ям.
- **`add_client_note`** — Додавання нової нотатки або запису про зустріч у профіль клієнта.
- **`get_team_structure`** — Відображення ієрархії співробітників та підлеглих для заданого менеджера.
- **`get_team_analytics`** — Збір статистики кількості клієнтів по відділу з розподілом по містах.
- **`create_client`** — Створення нової картки клієнта в CRM та закріплення її за поточним консультантом.
- **`onboard_new_client`** — Комплексний процес створення клієнта із миттєвим додаванням стартової нотатки.

## ▶️ Як запустити проєкт "з нуля"

### 1. Встановлення інструментів

- Docker та Docker Compose
- Node.js (v20+)
- Python 3.10+ (для аналітики)
- Локальний Ollama (для інференсу моделей)

### 2. Клонування репозиторію

```bash
git clone https://github.com/yurakhalus/corporate-llm-mcp-gateway.git
cd corporate-llm-mcp-gateway
```

### 3. Налаштування `.env` файлів

Створіть `.env` файл у кореневій папці на основі `.env.example`:
```bash
cp .env.example .env
```
У файлі `.env` **обов'язково** потрібно налаштувати ключі доступу:
1. `TWENTY_API_KEY`: Згенеруйте персональний токен доступу в налаштуваннях вашого профілю у Twenty CRM (Settings -> Developers -> API Keys).
2. `JWT_SECRET` / `OPEN_WEBUI_API_KEY`: Встановіть секретний ключ для підпису JWT токенів Open WebUI. Це необхідно для зв'язку між клієнтським інтерфейсом та шлюзом.
3. `AI_INFERENCE_URL`: Наприклад `http://host.docker.internal:11434`, якщо ви використовуєте Ollama, встановлену на хост-машині.

### 4. Запуск інфраструктури (Docker)

```bash
docker compose up -d
```
Це підніме бази даних (Postgres, Redis), Twenty CRM, AI Gateway, MCP Server, MCPO та Open WebUI.

### 5. Завантаження моделі в Ollama

Якщо ви використовуєте Ollama локально на Mac/PC:
```bash
ollama pull gemma4:e4b
# або
ollama pull llama3.1
```

### 6. Встановлення залежностей локальних скриптів

```bash
# Для бенчмарку
cd benchmarks
npm install

# Для Python-графіків
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## 🔌 API та Тестування (Приклади)

### 🚀 Запуск автоматизованого бенчмаркінгу

Скрипт автоматично прожене модель через набір тестів та запише телеметрію в лог-файл Gateway.

```bash
cd benchmarks
TEST_MODEL_NAME="llama3.1:latest" node run_benchmark.js
```

### 📊 Генерація графіків для дипломної роботи

Після запуску тестів згенеруйте порівняльні графіки:
```bash
cd benchmarks
source venv/bin/activate
python3 plot_metrics.py
```
Графіки з'являться у папці `benchmarks/plots/`.

## 🖱️ Інструкція для користувача (WebUI)

1. Відкрийте `http://localhost:8080` (Open WebUI).
2. Зареєструйте першого користувача (він стане адміністратором).
3. Оберіть завантажену модель (наприклад, `gemma4:e4b`).
4. Активуйте інструменти MCP (через кнопку у вигляді "плюса" або "інструмента" в чаті).
5. Запитайте щось у моделі, наприклад: *"Скільки у нас клієнтів в базі CRM?"*.

## 🧪 Проблеми і рішення

| Проблема | Рішення |
|----------|---------|
| `HTTP 401: Unauthorized` під час бенчмарку | Згенеруйте JWT токен в Open WebUI та додайте у файл середовища `OPEN_WEBUI_API_KEY`. |
| Точки на графіках злиплися на осі `X=0.0` | Не передавалися дані про токени. Переконайтеся, що в скрипті включено `stream_options: { include_usage: true }`. |
| Out of Memory (OOM) / Не вистачає VRAM | Використовуйте 4-bit квантовані моделі (наприклад, версії Q4_K_M) або зменшіть `AI_MAX_CONTEXT_LENGTH`. |
| Помилка `Model not found` | Переконайтеся, що назва моделі в змінній `TEST_MODEL_NAME` (в нижньому регістрі) повністю збігається з назвою у `ollama list`. |
